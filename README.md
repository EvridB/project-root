# Repair Requests Service

Веб-сервис для управления заявками в ремонтную службу. Реализованы роли диспетчера и мастера, атомарный захват заявки с защитой от гонок.

## Быстрый старт

1. Склонируйте репозиторий.
2. Запустите контейнеры:
docker-compose up

text
3. Откройте фронтенд: http://localhost:5173
4. Войдите под одним из тестовых пользователей: dispatcher, master1, master2

## Тестовые пользователи

- **Диспетчер**: dispatcher
- **Мастера**: master1, master2

## Проверка гонки (race condition)

1. Убедитесь, что заявка с ID=2 (из сидов) имеет статус `assigned` и назначена на master1.
2. Запустите скрипт:
bash race_test.sh

text
3. Ожидаемый результат: один запрос вернёт 200, второй — 409 Conflict.

## Автотесты

Запуск тестов (в контейнере backend или локально с установленными зависимостями):
pytest tests/
